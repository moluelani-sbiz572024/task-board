## Git 運用ルール

**コードに変更を加えたら、その都度 GitHub に自動で push すること。**

具体的な運用は以下の通り：

1. 意味のある1つの変更（機能追加・修正・設定変更など）が完了するたびに、都度コミットを作成する。
   - 複数の変更をまとめて1つの巨大なコミットにしない。
   - コミットメッセージは変更内容が分かる簡潔な日本語 or 英語で記述する。
2. コミット作成後、確認を挟まず `git push` まで自動で実行する。
   - ユーザーへの確認は不要（このルール自体が事前承認）。
3. push 先は `origin`（リモート: `https://github.com/moluelani-sbiz572024/task-board.git`）の、作業中のブランチとする。
   - 特に指定がなければ `main` ブランチで直接作業・push して構わない。
4. 以下の操作は必ず事前にユーザーに確認を取る（このルールの対象外・自動実行しない）:
   - `git push --force` / `git reset --hard` などの破壊的操作
   - リモートブランチの削除
   - コミット履歴の書き換え（rebase -i、amend で公開済みコミットを変更する等）
5. `.env` や認証情報など、秘密情報を含むファイルは絶対にコミット・push しない。
   - `git add` 前に `git status` で差分に不審なファイルが含まれていないか確認する。

### コミット時の注意

- コミットメッセージ末尾には Claude Code による生成であることを示す署名を含める:
  ```
  Co-Authored-By: Claude Sonnet 5 <noreply@anthropic.com>
  ```
- `git commit --no-verify` など、hook をスキップするオプションは使用しない。
